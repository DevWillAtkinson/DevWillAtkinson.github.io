Game Development

I used to be deeper into game development, but then I began to feel the technologies of the space were bloated. Many of my projects have been framework based as I tended to avoid large engines due to the overhead and lack of need for the features they provide.

Now I am more interested in de-bloating software and making small games with simple graphics frameworks that set up a simple way to send images to screens. Even now, I feel that sending images to screens can be more work than necessary.

When you're done setting up a device context, you're often a hundred or so lines of code in. I'm hoping there can be a better way and am currently researching alternative graphics technologies.
